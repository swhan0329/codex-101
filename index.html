01 Codex 에코시스템 개요

OpenAI Codex는 2025년 5월에 codex-1(o3 기반)으로 재출시된 후, 2026년 2월 현재 GPT-5.3-Codex까지 진화한 에이전틱 코딩 플랫폼입니다.

"단순 코드 생성 → 소프트웨어 엔지니어 팀원"

Codex는 코드 자동완성 도구가 아닙니다. 여러 파일을 동시에 수정하고, 테스트를 실행하고, 결과를 검증하며, 스스로 디버깅하는 자율 에이전트입니다.

06 Codex App (macOS)

2026년 2월 2일 출시. 에이전틱 코딩의 "커맨드 센터". 여러 AI 에이전트를 동시에 관리하는 macOS 전용 데스크톱 앱.

쓰레드 모드

모드 실행 위치 설명
Local 내 맥 현재 프로젝트 디렉토리에서 직접 작업
Worktree 내 맥 Git Worktree로 변경사항 격리. 병렬 작업에 적합
Cloud 클라우드 원격 클라우드 환경에서 실행

핵심 기능

—멀티에이전트 병렬 실행
—내장 Git 도구 (diff, commit, PR)
—Skills 지원 (CLI/IDE 공유)
—Automations (정기 작업 자동화)
—Voice Dictation (Ctrl+M)
—플로팅 팝아웃 창
—IDE Extension 실시간 동기
—이미지 입력 (Drag & Drop)
—내장 터미널 (쓰레드별)
—작업 중 절전 방지

13 MCP (Model Context Protocol)

AI 모델과 외부 도구(브라우저, DB, 파일시스템 등)를 연결하는 표준 프로토콜.

# MCP 서버 추가
codex mcp add playwright -- npx @playwright/mcp

# config.toml
[mcp_servers.playwright]
command = "npx"
args = ["@playwright/mcp"]
서버 용도 패키지
Playwright 웹 브라우저 제어 @playwright/mcp
Filesystem 파일 시스템 접근 @mcp/server-filesystem
GitHub GitHub API 연동 @mcp/server-github
Postgres DB 쿼리 실행 @mcp/server-postgres
Memory 세션 간 기억 유지 @mcp/server-memory

19 FAQ

Codex CLI / App / IDE / Cloud의 차이는?

모두 같은 Codex App Server 프로토콜을 사용하지만 인터페이스가 다릅니다. CLI는 터미널, App은 macOS 멀티에이전트, IDE는 VS Code 통합, Cloud는 브라우저에서 GitHub 연동.

무료로 사용할 수 있나요?

CLI는 오픈소스이지만, OpenAI API 또는 ChatGPT Plus($20/월) 이상 구독이 필요합니다.

Full Auto 모드가 안전한가요?

OS 수준 샌드박싱 + 네트워크 차단이 적용됩니다. Git으로 관리하면 언제든 되돌릴 수 있습니다.

GPT-5.3-Codex-Spark는 뭔가요?

2026.02 리서치 프리뷰로, 초당 1000+ 토큰의 초고속 코딩 모델입니다.

오프라인에서 사용 가능한가요?

MCP로 로컬 모델(Ollama 등)을 연결하면 가능하지만, 품질은 OpenAI 모델보다 떨어집니다.
