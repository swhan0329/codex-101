01 Codex 에코시스템 개요

OpenAI Codex는 2025년 5월에 codex-1(o3 기반)으로 재출시된 후, 2026년 2월 현재 GPT-5.3-Codex까지 진화한 에이전틱 코딩 플랫폼입니다.

"단순 코드 생성 → 소프트웨어 엔지니어 팀원"

Codex는 코드 자동완성 도구가 아닙니다. 여러 파일을 동시에 수정하고, 테스트를 실행하고, 결과를 검증하며, 스스로 디버깅하는 자율 에이전트입니다.

11 슬래시 명령어 & 단축키

플랫폼별 슬래시 명령어

아래 목록은 CLI / IDE / App 공식 문서를 교차 확인해 분리한 명령어입니다. 서로 이름이 비슷해도 동작이 다를 수 있습니다.

CLI (터미널)

CLI
/model 활성 모델 변경 (가능한 경우 reasoning effort 포함)
/permissions 승인/권한 정책 변경
/agent 활성 에이전트 스레드 전환
/plan 플랜 모드 전환 (선택적으로 인라인 프롬프트 포함)
/compact 대화 요약으로 컨텍스트 토큰 절약
/status 세션 설정/토큰 사용량 확인
/review 워크트리 코드 리뷰 실행
/feedback 진단 로그와 피드백 전송

CLI에는 이외에도 /personality, /experimental, /init, /mention, /mcp, /diff, /fork, /resume, /new, /quit 등이 있습니다. /approvals는 별칭으로 동작하지만 팝업 목록에는 표시되지 않습니다.

14 MCP (Model Context Protocol)

AI 모델과 외부 도구(브라우저, DB, 파일시스템 등)를 연결하는 표준 프로토콜.

# MCP 서버 추가
codex mcp add playwright -- npx @playwright/mcp

# config.toml
[mcp_servers.playwright]
command = "npx"
args = ["@playwright/mcp"]
서버 용도 패키지
Playwright 웹 브라우저 제어 @playwright/mcp
Filesystem 파일 시스템 접근 @mcp/server-filesystem
GitHub GitHub API 연동 @mcp/server-github
Postgres DB 쿼리 실행 @mcp/server-postgres
Memory 세션 간 기억 유지 @mcp/server-memory

📋 워크플로 및 계획

기본부터 시작, 반복적 빌드

복잡한 작업은 단순하게 시작하고 점진 확장. Codex에게 결과를 검증(Verify)하도록 요청하면 최적 성능.

/plan(/plan-mode) → 실행 → /review

/plan(CLI) 또는 /plan-mode(App)로 계획을 세운 뒤 작업을 실행하고, 마지막에 /review로 결과를 점검하세요.

큐잉 & 병렬 처리

아이디어가 떠오를 때마다 Codex에 큐를 쌓아 병렬 처리. 여러 에이전트를 동시 투입하면 거의 모든 것을 달성 가능.

AGENTS.md 적극 업데이트

🔧 스킬 및 도구 통합

추천 스킬

프로젝트 성격에 맞는 스킬을 우선 사용하세요. 예: Playwright(브라우저 자동화), code-review(리뷰 품질 강화), tdd(테스트 우선 개발).

워크트리 & 백그라운드

워크트리 모드로 변경 격리, 병렬 작업 시 충돌 방지. 백그라운드 터미널에서 테스트/린트/Git 실행. 결과를 인박스로 전달.

IDE 싱크 & Git 리뷰

현재 보고 있는 파일에 자동 컨텍스트 제공. 내장 Git 리뷰로 Diff 확인, hunk 스테이징/되돌리기, 인라인 댓글 가능.

💻 앱 / CLI / 웹 팁

CLI 우선 빌드

에이전트가 검증하기 쉬운 CLI부터 시작하고, UI 등 다른 표면을 추가. 코드가 일회성이라면 아이디어 테스트 후 브랜치 폐기.

앱 편의 기능

여러 프로젝트/스레드를 하나의 앱에서 관리. Ctrl+M으로 음성 입력. /persona로 응답 스타일 조정.

웹 활용

Codex Web에서 GitHub를 연결하고 자동 PR 리뷰를 활성화하면 비동기 검증 흐름을 만들기 좋습니다.

20 FAQ

Codex CLI / App / IDE / Web의 차이는?

모두 같은 Codex App Server 프로토콜을 사용하지만 인터페이스가 다릅니다. CLI는 터미널, App은 macOS 멀티에이전트, IDE는 VS Code 통합, Cloud는 브라우저에서 GitHub 연동.

무료로 사용할 수 있나요?

네! ChatGPT 무료 계정으로도 Codex CLI, App, Web 모두 한시적 무료 접근이 가능합니다 (5시간당 약 10~30회). 안정적 사용을 위해서는 Plus($20/월) 이상을 권장하며, API Key 종량제도 지원됩니다.

Full Auto 모드가 안전한가요?

OS 수준 샌드박싱 + 네트워크 차단이 적용됩니다. Git으로 관리하면 언제든 되돌릴 수 있습니다.

GPT-5.3-Codex-Spark는 뭔가요?

2026.02 리서치 프리뷰로 공개된 텍스트 중심 코딩 모델입니다. 빠른 반복 작업에 초점을 둔 옵션이며, 최신 제공 범위는 플랜/문서 기준으로 확인하세요.

오프라인에서 사용 가능한가요?

Codex CLI는 오픈소스(Rust)이며, 타사 모델 프로바이더(Ollama 등)를 연결할 수 있지만, 품질은 OpenAI 모델보다 떨어집니다.
